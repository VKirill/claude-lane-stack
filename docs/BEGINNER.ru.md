# Гайд для новичков — Claude Lane Stack

**Не нужно быть гиком по мультиагентам.**  
Система — как маленький завод: ты общаешься с одним прорабом, он раздаёт задания, готовое попадает в основную ветку `main`.

English: [BEGINNER.md](BEGINNER.md)

---

## 1. Что такое «оркестрация» простыми словами?

| В жизни | У нас |
|---------|--------|
| Ты — **хозяин** | Человек |
| Нанял **прораба** | Claude Code, агент `dev-orchestrator` |
| Прораб зовёт **мастеров и проверяющего** | AGY, Grok, Codex (по желанию) |
| Задания на **карточках**, а не криком | Файлы в `.agents/runs/` |
| Готовое на **основной склад** | Ветка git **`main`** |

**Оркестрация** = прораб сам решает, кто пишет код, проверяет результат и кладёт его в `main`.  
Ты **не** ведёшь пять чатов и **не** мержишь ветки руками.

## 2. Словарик

| Слово | Простыми словами | Когда важно |
|-------|------------------|-------------|
| **Агент** | ИИ, который умеет читать/писать код | Всегда |
| **PM / оркестратор** | «Прораб» (`dev-orchestrator`) | С ним ты в основном и говоришь |
| **Lane (линия)** | Тип работника: быстрый / основной / ревью | Когда настраивают AGY/Grok/Codex |
| **Claude Code** | Программа Anthropic в терминале | **Обязательна** |
| **Задача / контракт** | YAML-файл: что сделать, какие файлы, как проверить | PM создаёт, работники читают |
| **`.agents/runs/`** | Папка текущих работ (цех) | Когда «делай / реализуй» |
| **`docs/plans/`** | Стратегия, SEO, длинные планы | Сначала исследование, потом «в код» |
| **`main`** | Главная ветка = «официальный» код | Конец успешной работы |
| **Worktree** | Отдельная копия проекта, чтобы не мешать друг другу | PM включает при параллели |
| **Merge** | Свести работу в `main` | **Делает PM**, не ты |
| **Онбординг** | Первый паспорт проекта (CLAUDE.md, доки) | Один раз на репозиторий |
| **Cold start / resume** | Продолжить после перерыва | На следующее утро / новый чат |

## 3. Команды: что значат и когда запускать

В **терминале**, обычно из папки **твоего** проекта.

### А. Один раз на компьютере

| Команда | Зачем | Когда |
|---------|--------|------|
| `git clone …/claude-lane-stack.git` | Скачать стек | Первый раз |
| `./install.sh` | Поставить агентов и инструменты | Первый раз |
| `export PATH="$HOME/.agents/bin:$PATH"` | Чтобы находились команды | Каждый новый терминал (или прописать в `~/.bashrc`) |

### Б. Один раз на каждый проект

| Команда | Зачем | Когда |
|---------|--------|------|
| `cd /путь/к/проекту` | Зайти в **свой** проект | Перед работой |
| `agents-doctor --apply .` | Посмотреть, какие CLI есть, записать профиль | Первый раз / после установки нового CLI |
| `/project-onboard` в Claude | Codex заполнит CLAUDE.md и первичную доку | Первый раз в проекте |
| `claude --agent dev-orchestrator` | Запустить **прораба** | Каждый раз, когда нужен завод |

### В. Каждый день

| Что вводишь | Что происходит | Когда |
|-------------|----------------|------|
| Обычный запрос: *«добавь кнопку входа»* | PM планирует, раздаёт, проверяет, мержит | Обычная работа |
| `/resume-project` или `resume-project .` | Кратко: сейчас / блок / дальше | **После перерыва**, новый чат — *не* шаг №1 установки |
| «Зависло / stall» | Ищут «замолчавшего» работника | Долго нет ответа |

### Г. Редко руками (обычно PM)

| Команда | Смысл |
|---------|--------|
| `run-board` | Табло задач |
| `wt-create` / `wt-merge-main` | Изоляция и merge в main (**только PM**) |
| `check-owns-paths` | Не лезли ли в чужие файлы |
| `lane-stall-check` | Кто завис |

> **Запомни три шага:**  
> 1) `./install.sh`  
> 2) `agents-doctor --apply .` в проекте  
> 3) `claude --agent dev-orchestrator` и пиши по-человечески

## 4. Путь новичка

### День 0 — установка

1. Поставь Claude Code (остальные CLI — по желанию).  
2. Склонируй стек и `./install.sh`.  
3. Новый терминал → `export PATH=…`  
4. `cd` в **свой** проект → `agents-doctor --apply .`  
5. `claude --agent dev-orchestrator` → `/project-onboard`  

### День 1 — первая задача

1. Снова PM.  
2. Одна **маленькая** цель.  
3. Не мержи сам — дождись «готово» от PM.  

### День 2 — после сна

Чат «забыл» контекст — это нормально.

```bash
claude --agent dev-orchestrator
```

Потом **`/resume-project`** (или `resume-project .`) → **Сейчас / Блок / Дальше** → продолжай.

> **Сноска:** *Cold start* = «начали чат заново, старая переписка не в голове».  
> `/resume-project` — шпаргалка на этот момент, а не ежедневный ритуал и не первый шаг установки.
